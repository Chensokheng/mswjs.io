---

Request query parameters are stored in the `req.url` object, which is an instance of the `URL` class, and can be accessed under the `req.url.searchParams` property. The value of that property is a [`URLSearchParams`](https://developer.mozilla.org/en-US/docs/Web/API/URLSearchParams) instance that contains all query parameters. Storing the query parameters in a `URL` delegates the parameters parsing and normalization to the standard browser API, instead of being a part of the library.

---

## Single parameter

```js highlights=2
// Given a "GET /products?id=1" request...
rest.get('/products', (req, res, ctx) => {
  // ...the value of the "id" parameter is "1".
  const productId = req.url.searchParams.get('id')

  return res(
    ctx.json({
      productId,
    }),
  )
})
```

> Learn more about the [`URLSearchParams.get()`](https://developer.mozilla.org/en-US/docs/Web/API/URLSearchParams/get) method.

## Multiple parameters

The values of multiple query parameters under

Call `req.url.searchParams.getAll()` method to access all values assigned to a parameter.

```js highlights=2
// Given a "GET /products&id=1&id=2&id=3" request...
rest.get('/products', (req, res, ctx) => {
  // ...the values of the "id" parameter is a list
  // of all provided values: ['1', '2', '3'].
  const productIds = req.url.searchParams.getAll('id')

  return res(
    ctx.json({
      productIds,
    }),
  )
})
```

> Learn more about the [`URLSearchParams.getAll()`](https://developer.mozilla.org/en-US/docs/Web/API/URLSearchParams/getAll) method.
